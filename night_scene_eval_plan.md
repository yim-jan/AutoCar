# 夜间+逆光场景评测方案

## 1. 场景定义
- **夜间**：图片平均亮度 < 60（0-255），通过 OpenCV 计算灰度图均值判定
- **逆光**：图片上半部分（天空）亮度 > 下半部分（地面）亮度的 1.5 倍，模拟对向强光源

## 2. 评测指标
- 困难场景平均检出数：对比正常场景，越低说明模型对光照越敏感
- 低置信度检出率（< 0.5）：越高说明模型在困难场景下"犹豫"不决
- 车辆/行人检出分布：判断是否对特定类别更敏感
- **重点关注漏检，而非误检** — 夜间漏掉行人的代价远大于误报

## 3. 数据筛选
- 来源：COCO val2017 第201-300张（不与校准集重叠）
- 亮度计算：cv2.cvtColor + cv2.mean
- 逆光判定：上下半区域亮度比值
- 阈值可配置，方便后续调优

## 4. 预期输出
| 项目 | 内容 |
|------|------|
| 场景分布 | 夜间 X 张 / 逆光 Y 张 / 困难场景合计 Z 张 |
| 检测对比 | 困难场景 vs 正常场景 平均检出数 |
| 可视化样本 | 保存前10张困难场景的检测框图片 |
| 改进建议 | 是否需要增加训练数据、调整置信度阈值 |

## 5. 工程扩展方向
- 引入 COCO 标注真值，计算真实 mAP 而非仅输出分布统计
- 按光照强度分档（极暗/暗/正常/过曝），输出分档评测报告
- 对接标注团队，将误检漏检图片回流到训练集
